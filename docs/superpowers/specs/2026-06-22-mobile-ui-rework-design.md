# Дизайн: переписать UI под телефон + вырезать код энкодера/экрана

Дата: 2026-06-22
Ветка: `mobile-ui-rework` (форк от `main`, базовый коммит — рабочая прошивка)
Режим: автономный (решения приняты Claude, согласование — после возврата пользователя)

## Цель

1. Сделать веб-интерфейс удобным для управления **с телефона (iPhone, Safari)**.
2. Вырезать «мёртвый» код энкодера и OLED-дисплея (отключены флагами, физически сняты).
3. **Состояние концевиков (TOP/A/B/BOT) видно ВСЕГДА**, при любом экране — для отладки.

## Контекст

- Прошивка ESP8266, headless (без OLED/энкодера), управление по WebSocket (`ws://<ip>/ws`).
- Протокол уже поддерживает полный цикл без энкодера: `command` (start_charging, pusk,
  abort, new_cycle) + `direct_set` (объём/диаметр шприцов, время дозирования, шаг винта, сон).
- Цикл состояний: PARKING → (конц. TOP) → PARKED → CHARGING → (конц. A&B) → CHARGED →
  DOSING → (конц. BOT/таймер) → DONE → (Новый цикл) → PARKING.
- Текущий веб — отладочный пульт: ASCII-копия OLED, эмулятор энкодера, debug-switches, лог.

## Решения (автономные)

### Архитектура
- **Мобильный UI — чисто фронтенд** (data/index.html, app.js, app.css), на существующем
  протоколе. Это даёт основную ценность с нулевым риском для рабочей прошивки.
- **Живая отладка концевиков** требует маленькой правки прошивки: поле `rawSwitches`
  (реальные показания `ILimitSwitches::read()` каждый tick) в JSON-состоянии + бродкаст
  при изменении. Иначе `state.switches` частично сценарные.

### Мобильный UI (data/)
- Одностраничный, mobile-first, тёмная тема, крупные кнопки (touch).
- **Залипающая шапка**: заголовок, статус связи и **всегда видимая панель концевиков**
  (TOP/A/B/BOT, живые LED из `rawSwitches`). Видна на любом экране, не гаснет.
- Центр — карточка стадии, меняется по `screen`:
  - PARKING — «ожидание концевика TOP», индикатор.
  - PARKED — выбор шприца A/B (пресет + диаметр), кнопка «Старт зарядки».
  - CHARGING — «ожидание A и B».
  - CHARGED — время дозирования, расчётные потоки A/B, кнопка «ПУСК».
  - DOSING — крупный прогресс %, прошло/всего, объёмы A/B, кнопка «СТОП».
  - DONE — итог, кнопка «Новый цикл».
- Настройки (сворачиваемый блок): шаг винта, таймаут сна. Через `direct_set` (в любой момент).
- Управление: `command` + `direct_set`. Эмуляция энкодера и ASCII-OLED — убраны.

### Вырезание кода (отдельный коммит, с проверкой сборки)
- Удалить OLED: `OledRenderer`, `Fonts.h`, `FontConfig`, `ui/screens/*`; убрать из main.cpp
  (g_oled, showSplash, render, USE_OLED, I2C/Wire — реальных I2C-устройств нет).
- Удалить энкодер: `EncoderInput`; убрать g_encoder, poll-loop, `handleEncoder` и тип
  сообщения `encoder` из WsProtocol; убрать `EncoderInput*` из сигнатур.
- `StateMachine`: убрать `onEvent` + все `handle*`/`adjust*`/`navWrap`/breadcrumb
  (энкодерная навигация). Оставить `transitionTo` + `tick` (команды + концевики).
- Экраны `SERVICE_*` и `FONT`/`circularNav` — убрать (без OLED-навигации не нужны;
  настройки идут через `direct_set`). Команды enter/exit_service — убрать.
- Редактирование пресетов шприцов сохранить через новый `direct_set`-путь
  `set_preset` (idx, vol, diam), чтобы конфигурируемость не потерялась.

### Сознательно выкинуто (YAGNI без OLED)
- Выбор шрифта (`fontIndex`, SERVICE_FONT) — нет дисплея.
- `circularNav` — нет курсорной навигации.
- Эмулятор энкодера, ASCII-OLED, debug_switch (на реальном железе не нужно — есть
  живые `rawSwitches`).

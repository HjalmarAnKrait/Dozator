# Hardware – GPIO Map

> **Headless-конфигурация.** OLED и энкодер удалены, их пины переназначены на мотор
> и концевики. Управление и статус — через веб-интерфейс (точка доступа WiFi `dozator`).

## 2.1 Текущая привязка (активна)

| Функция          | Wemos | GPIO  | Режим / подключение                                  |
|------------------|-------|-------|------------------------------------------------------|
| Stepper STEP     | D2    | GPIO4 | Output → A4988 STEP                                   |
| Stepper DIR      | D1    | GPIO5 | Output → A4988 DIR                                    |
| Микрошаг MS1     | D7    | GPIO13| Output HIGH (1/16)                                    |
| Микрошаг MS2     | D6    | GPIO12| Output HIGH                                           |
| Микрошаг MS3     | D5    | GPIO14| Output HIGH                                           |
| Концевик TOP     | RX    | GPIO3 | `INPUT_PULLUP`, на **GND** (active-LOW)               |
| Концевик A       | D3    | GPIO0 | `INPUT_PULLUP`, на **GND** (active-LOW) ⚠️ boot        |
| Концевик B       | D4    | GPIO2 | `INPUT_PULLUP`, на **GND** (active-LOW) ⚠️ boot        |
| Концевик BOT     | D0    | GPIO16| `INPUT_PULLDOWN_16`, на **3.3В** (active-HIGH)        |
| Кнопка СТОП      | D8    | GPIO15| `INPUT`, на **3.3В** (active-HIGH) ⚠️ boot             |
| EN драйвера      | —     | —     | на драйвере: RST+SLP→+5V, ENABLE не заведён (всегда вкл.) |

### ⚠️ Важно: разная полярность концевиков
GPIO16 (BOT) и GPIO15 (кнопка) **не имеют внутренней подтяжки вверх**, поэтому:
- **TOP, A, B → на GND** (INPUT_PULLUP, срабатывание = LOW).
- **BOT и кнопка СТОП → на 3.3В** (срабатывание = HIGH).

Не перепутай: если BOT или кнопку завести на GND — работать не будут.

### ⚠️ Boot-пины (состояние при ВКЛЮЧЕНИИ)
- **A (D3/GPIO0), B (D4/GPIO2)** — концевики должны быть **разомкнуты** при старте
  (иначе плата не загрузится). Т.е. в момент включения шприцы не должны прижимать A/B.
- **Кнопка СТОП (D8/GPIO15)** — **не нажата** при включении.

### Прочее
- **TOP на RX (GPIO3):** приём по Serial больше не работает (вывод логов по TX — работает).
- **A4988:** RESET и SLEEP соединены и на +5V; ENABLE не заведён (драйвер всегда включён).
  Микрошаг задаёт МК (MS1/MS2/MS3 = HIGH → 1/16), DIP не нужен.

## 2.2 Схема подключения драйвера A4988

```
Wemos D1 mini        A4988
   D2 (GPIO4)  ────── STEP
   D1 (GPIO5)  ────── DIR
   D7 (GPIO13) ────── MS1  ┐
   D6 (GPIO12) ────── MS2  ├ 1/16 (все HIGH)
   D5 (GPIO14) ────── MS3  ┘
   GND         ────── GND
   +5V  ───┬────────  RESET
           └────────  SLEEP     (RST и SLP соединены, на +5V)
                      ENABLE    (не подключать — драйвер включён)
   +5V         ────── VDD
   +12V        ────── VMOT   (+ конденсатор 100мкФ по VMOT/GND)
   A4988 1A/1B/2A/2B ── обмотки мотора
```

## 2.3 Концевики и кнопка

```
   TOP  → RX (GPIO3) ─┐
   A    → D3 (GPIO0) ─┤  вторые контакты → GND   (active-LOW)
   B    → D4 (GPIO2) ─┘

   BOT      → D0 (GPIO16) ─┐  вторые контакты → 3.3В  (active-HIGH!)
   Кнопка   → D8 (GPIO15) ─┘
```

Концевики нормально-разомкнутые (NO). Для NC — инвертировать логику в `RealSwitches::tick()`.

## 2.4 История: снятая периферия

OLED (SSD1306, I2C на D1/D2) и квадратурный энкодер (D5/D6/D7) были в ранней версии,
затем удалены — устройство headless, управление через веб. Их пины переназначены на
мотор и концевики (таблица §2.1).
